# 自驱力与自主探索

[中文](SELF_DRIVE.md) | [English](SELF_DRIVE.en.md)

## 目标

灵舟的自驱力不是定时任务，也不是机械巡检。它的作用是：在没有外部消息时，基于当前记忆状态、失败历史和预测误差，判断系统是否应该继续探索、整理知识、修正自身，还是暂时保持安静。

## 三类核心信号

灵舟的自主探索主要由三类内在信号驱动：

| 信号 | 含义 | 典型来源 |
|------|------|----------|
| Novelty | 新颖度，近期是否接触了新的对象、知识或路径 | 新文件、新网页、新任务、新记忆节点 |
| Learning Progress | 学习进度，最近是否真正获得了新能力或闭环结果 | 任务完成、反思结晶、失败后修复成功 |
| Surprise | 惊异度，运行结果与预期偏差是否明显 | 工具报错、探针异常、配置缺失、行为循环 |

这三类信号不会直接替代判断层，而是先进入感知链路，成为后续决策的输入。

## 在认知循环中的位置

灵舟的主链路是：

```text
感知 -> 自驱力 -> 判断 -> 执行 -> 反思
```

其中自驱力负责回答两个问题：

1. 现在有没有必要主动做点什么。
2. 如果要动，优先探索、验证、整理，还是等待。

它不会直接执行工具，而是把驱动信号交给判断层，由判断层统一决定是否调用工具、是否回复用户、是否推进任务。

## 何时会触发自主行为

常见触发场景：

- 当前没有新的用户消息，但记忆中存在高价值未整理线索
- 连续失败后需要主动反思或补证据
- 探针发现外部状态变化，需要重新评估
- 长时间空闲，但系统判断还有值得探索或压缩的内容

常见不触发场景：

- 用户正在等待明确回复，本轮应优先完成当前交互
- 最近已连续多轮同类探索且没有新增证据
- 当前环境不具备继续推进的关键外部条件

## 与记忆系统的关系

自驱力强弱直接受记忆状态影响：

- 工作记忆压力过高时，更倾向于整理和压缩，而不是继续扩张
- 语义记忆命中质量低时，更倾向于补证据，而不是直接下结论
- 情节记忆中若连续记录相同失败，会提高 surprise，促使策略切换

因此，自驱力不是孤立模块，而是记忆系统的消费端和放大器。

## 关键配置

相关配置主要位于 `lingzhou.json`：

- `loop.max_idle_gap`：空闲时最长等待间隔
- `loop.active_idle_gap`：有活跃任务时的等待间隔
- `thresholds.curiosity_idle_task`：空闲探索阈值
- `thresholds.curiosity_idle_min_cycles`：最少连续空闲轮次
- `memory.semantic_decay_lambda`：语义记忆衰减率
- `memory.chat_crystallize_every`：聊天结晶频率

这些配置决定了灵舟是在空闲时更保守，还是更主动。

## 当前边界

目前自驱力模块仍受以下边界约束：

- 自主探索必须服从任务边界与硬约束，不能绕开安全规则
- 高成本操作仍需经过判断层，不会因为“好奇”而直接执行危险动作
- 当已有用户问题待回复时，优先完成当前交互，而不是转入无关探索
