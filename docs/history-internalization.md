# 历史经验在 lingzhou 中的内化边界

## 核心原则

lingzhou 不直接以“旧宿主导入物”的身份消费历史材料。
历史只能以 **当前运行时可理解、可调用、可维护** 的原生结构进入系统。

换句话说：

- 保留能力与经验
- 丢掉宿主壳、迁移过程、来源中心叙事
- 让当前运行时只面对 `lingzhou` 自己的概念

## lingzhou 的原生承接面

### 1. 身份层
进入：
- `BOOTSTRAP.md`
- `IDENTITY.md`
- `SOUL.md`
- `facts["soul:*"]`

只保留：
- 主体性
- 价值基线
- 对用户的对象感
- 长期行为原则

不保留：
- 旧宿主名字
- 迁移口吻
- 过渡期策略说明

### 2. 任务层
进入：
- `tasks`
- `signals`
- `failures`

只保留当前运行时仍然有效的目标、阻塞、证据和下一步。

不保留：
- 一次性迁移任务
- 旧宿主对照任务
- 过渡阶段的导入检查单

### 3. 语义记忆层
历史材料只允许以内化后的三类节点存在：

- `identity_anchor`
  - 身份锚点、时间锚点、长期主线
- `historical_principle`
  - 由历史经验沉淀出的长期原则、纠偏、结构性认知
- `historical_episode`
  - 仍有参考价值的历史片段、案例、故障现场、实践样本

不允许继续保留：
- `*_source`
- `migration_*`
- `ocmig_*`
- 以导入过程命名的节点种类或 id

## 原始历史资料放哪里

原始资料与导入证据只保留在：

- `~/.lingzhou/imports/`
- `archive/`
- `~/.lingzhou/backups/`

这些地方属于：
- 追溯
- 审计
- 对照
- 恢复

不属于 runtime。

## 判断标准

如果某条内容离开“旧宿主名字 / 导入过程 / 路径对照”就失去意义，
那它还没有被内化，应该继续留在 archive，而不是进入 lingzhou runtime。

如果某条内容可以直接回答下面问题，它才算已内化：

- 这对当前判断有什么帮助？
- 这对当前任务推进有什么帮助？
- 这对长期身份、原则、能力有什么帮助？

## 一句话定义

内化不是把旧东西搬进来。
内化是把历史经验压缩成 lingzhou 自己的身份、原则、片段与能力，让 runtime 不再依赖来源叙事。
